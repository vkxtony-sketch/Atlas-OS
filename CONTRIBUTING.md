# Contributing to Atlas OS

Thanks for considering contributing to Atlas OS. This project is an experimental multi-agent operating architecture focused on AI orchestration systems.

## How to Contribute

### 1. Issues
- Use GitHub Issues to report bugs or request features
- Clearly describe the problem or proposal
- Include context and examples when possible

### 2. Branching
- Create feature branches from `main`
- Use descriptive branch names like `agent-consensus-improvement` or `memory-system-refactor`

### 3. Code Style
- Keep modules small and composable
- Prefer clarity over complexity
- Document agent behavior and decision flows

### 4. Architecture Contributions
Atlas OS is structured around:
- Executive AI
- Agent Teams
- Consensus Engine
- Execution Sandbox

Contributions should align with or extend these systems.

### 5. Pull Requests
- Keep PRs focused and small
- Describe what changed and why
- Include test cases or simulations when possible

## Vision Reminder
Atlas OS aims to become a unified orchestration layer for autonomous AI systems. Contributions should improve coordination, reliability, or intelligence of the system.
